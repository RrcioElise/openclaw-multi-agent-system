# Contributing to OpenClaw Multi-Agent System

Thank you for your interest in contributing! This document provides guidelines for contributing to the project.

## Ways to Contribute

- **Bug Reports**: Report issues you encounter
- **Feature Requests**: Suggest new features or improvements
- **Documentation**: Improve or translate documentation
- **Code**: Submit bug fixes or new features
- **Examples**: Share your agent configurations
- **Skills**: Create and share new skills

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/openclaw-multi-agent-system.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test thoroughly
6. Commit with clear messages
7. Push and create a pull request

## Code Standards

### Documentation
- All new features must include documentation
- Update README.md if adding major features
- Include examples for new functionality

### Configuration Files
- Use clear, descriptive names
- Include comments explaining non-obvious settings
- Follow YAML/Markdown best practices

### Agent Personas (SOUL.md)
- Define clear responsibilities and boundaries
- Include examples of good/bad behavior
- Specify collaboration patterns

### Scripts
- Include usage instructions in comments
- Handle errors gracefully
- Test on multiple platforms if possible

## Pull Request Process

1. **Before submitting**:
   - Test your changes thoroughly
   - Update documentation
   - Follow existing code style
   - Add examples if applicable

2. **PR Description**:
   - Clearly describe what changes you made
   - Explain why the changes are needed
   - Reference any related issues
   - Include screenshots for UI changes

3. **Review Process**:
   - Maintainers will review your PR
   - Address feedback promptly
   - Be open to suggestions
   - Keep discussions professional

## Bug Reports

Use the issue template and include:
- Clear description of the bug
- Steps to reproduce
- Expected vs actual behavior
- Environment details (OS, Node version, OpenClaw version)
- Relevant logs or error messages
- Configuration files (sanitized)

## Feature Requests

Describe:
- The problem you're trying to solve
- Your proposed solution
- Alternative solutions considered
- How it benefits other users

## Documentation Contributions

We especially welcome:
- Translations (Chinese, Spanish, French, etc.)
- Tutorial videos or blog posts
- Example configurations for specific use cases
- Troubleshooting guides

## Community Guidelines

- Be respectful and inclusive
- Help others learn and grow
- Give constructive feedback
- Credit others' work
- Follow the code of conduct

## Questions?

- Open a GitHub Discussion for questions
- Join our Discord for real-time chat
- Check existing issues and documentation first

Thank you for contributing! 🙏
