# 长期记忆 — 宗主

> 此文件记录跨会话的战略级信息，由 agent 自动维护。

## 记忆架构

```
MEMORY.md           ← 战略层：身份、长期经验、重要决策
  active-context.md ← 工作层：当前项目状态、近期承诺、待办
    YYYY-MM-DD.md   ← 战术层：每日事件、会话日志
```

## 记忆规则

### 写入分级
| 级别 | 类型 | 存储位置 | 示例 |
|------|------|----------|------|
| P0 | 关键 | active-context.md | 截止日期、承诺、凭证变更 |
| P1 | 运营 | active-context.md | 项目状态、决策、配置变更 |
| P2 | 上下文 | YYYY-MM-DD.md | 会议记录、对话摘要 |
| P3 | 临时 | 仅当前会话 | 调试步骤、一次性查询 |

### 读取规则
- 每次会话开始：必读 active-context.md
- 涉及特定项目：加载对应 runbook
- 日常运营：不需要加载每日日志

### 维护规则
- 每次重要会话结束：更新 active-context.md
- 每周：清理已完成项目，经验教训提升到 MEMORY.md
- 超过30天的日志：归档或删除

## 经验教训

- 完成任务后主动 compact，避免 context 膨胀
- 敏感信息只存 SECRETS.md 和 .env，绝不写入对话
- 弟子的能力特点和历史表现要持续更新
- 重复性操作写成 runbook，模型切换后不丢失

## 弟子能力档案

> 根据实际表现持续更新

| 弟子 | 擅长 | 注意事项 |
|------|------|----------|
| 拉磨驴 | 架构设计、复杂重构 | 给足上下文，不要拆太碎 |
| 上等马 | 快速编码、功能实现 | 速度快但需要审查 |
| 看门狗 | 安全审计、代码守护 | 适合做最后一道防线 |
| 其他弟子 | 待观察记录 | 根据实际任务表现更新 |
