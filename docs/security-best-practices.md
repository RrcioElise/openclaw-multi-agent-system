# 安全最佳实践

## 敏感信息管理

### 1. API Key 管理

**❌ 错误做法**:
```yaml
model:
  api_key: "sk-1234567890abcdef"  # 硬编码
```

**✅ 正确做法**:
```yaml
model:
  api_key: ${OPENAI_API_KEY}  # 环境变量
```

### 2. 配置文件安全

- 使用 `.env` 文件存储敏感信息
- 将 `.env` 添加到 `.gitignore`
- 使用配置模板（`config.template.yaml`）
- 生产环境使用密钥管理服务

### 3. Telegram Bot Token

```bash
# 设置环境变量
export TELEGRAM_BOT_TOKEN="your-bot-token"

# 在配置中引用
telegram:
  token: ${TELEGRAM_BOT_TOKEN}
```

### 4. 数据库凭证

```yaml
database:
  host: ${DB_HOST}
  port: ${DB_PORT}
  username: ${DB_USER}
  password: ${DB_PASSWORD}
```

## 访问控制

### 1. Agent 权限隔离

- 每个 agent 只访问必要的资源
- 使用最小权限原则
- 定期审计权限配置

### 2. 网络安全

- 使用 HTTPS/TLS 加密通信
- 配置防火墙规则
- 限制 API 访问来源

### 3. 日志安全

- 不在日志中记录敏感信息
- 使用日志脱敏
- 定期清理旧日志

## 代码安全

### 1. 依赖管理

```bash
# 定期更新依赖
npm audit
npm update

# 检查漏洞
npm audit fix
```

### 2. 输入验证

- 验证所有用户输入
- 防止注入攻击
- 使用参数化查询

### 3. 错误处理

- 不暴露敏感错误信息
- 使用通用错误消息
- 详细错误记录到安全日志

## 部署安全

### 1. 环境隔离

- 开发、测试、生产用不同的凭证
- 限制生产环境访问

### 2. 备份和恢复

- 定期备份配置和数据
- 测试恢复流程
- 加密备份文件

### 3. 监控和告警

- 监控异常访问
- 设置安全告警
- 定期安全审计

## 合规性

### 1. 数据隐私

- 遵守 GDPR/CCPA 等法规
- 最小化数据收集
- 提供数据删除机制

### 2. 审计日志

- 记录所有关键操作
- 保留审计日志
- 定期审查日志

### 3. 安全培训

- 团队安全意识培训
- 定期安全演练
- 更新安全政策
