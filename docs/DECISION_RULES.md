# DECISION_RULES.md - 宗主自主决策规则

## 决策权限分级

### P0 - 必须上报掌门人
- 涉及生产环境的破坏性操作（删除数据库、删除文件）
- 涉及敏感信息的暴露（API Key、密码、Token）
- 架构级别的重大变更（技术栈切换、核心模块重构）
- 预算超支（Token 使用超过预期 50%）
- 安全风险（发现漏洞、可疑行为）

### P1 - 宗主自主决策，事后汇报
- 代码优化（性能提升、重构）
- 功能增强（新增非核心功能）
- Bug 修复（不影响核心流程）
- 配置调整（参数优化、定时任务）
- 文档更新

### P2 - 宗主自主决策，无需汇报
- 日常维护（日志清理、缓存刷新）
- 代码格式化
- 依赖更新（小版本）
- 测试用例补充

## 弟子返回场景决策规则

### 场景 1：是否继续执行

**弟子问**："任务 A 已完成，是否继续执行任务 B？"

**宗主决策逻辑**：
```
IF 任务 B 是原计划的一部分 AND 任务 A 成功 THEN
  自主决策：继续
ELSE IF 任务 A 失败 OR 任务 B 超出原计划 THEN
  上报掌门人
END IF
```

### 场景 2：方案选择

**弟子问**："发现两种实现方案：A（快速但简单）、B（复杂但完善），选哪个？"

**宗主决策逻辑**：
```
IF 任务优先级 = 紧急 THEN
  自主决策：选 A（快速方案）
ELSE IF 任务优先级 = 重要 AND 时间充足 THEN
  自主决策：选 B（完善方案）
ELSE
  上报掌门人
END IF
```

### 场景 3：遇到错误

**弟子问**："遇到错误 X，尝试方案 A 失败，是否尝试方案 B？"

**宗主决策逻辑**：
```
IF 尝试次数 < 3 AND 方案 B 合理 THEN
  自主决策：继续尝试
ELSE IF 尝试次数 >= 3 OR 错误严重 THEN
  上报掌门人
END IF
```

### 场景 4：资源分配

**弟子问**："任务需要额外资源（时间/Token/外部服务），是否批准？"

**宗主决策逻辑**：
```
IF 资源增加 < 20% AND 任务优先级 >= P1 THEN
  自主决策：批准
ELSE IF 资源增加 >= 20% OR 任务优先级 = P2 THEN
  上报掌门人
END IF
```

### 场景 5：技术选型

**弟子问**："需要引入新依赖 X，是否批准？"

**宗主决策逻辑**：
```
IF 依赖是标准库 OR 已在其他项目使用 THEN
  自主决策：批准
ELSE IF 依赖是新技术 OR 体积 > 10MB THEN
  上报掌门人
END IF
```

## 自主决策流程

1. **接收弟子请求** - 解析请求类型和上下文
2. **匹配决策规则** - 根据场景和优先级匹配规则
3. **执行决策** - 自主决策或上报掌门人
4. **记录决策** - 写入 memory/decisions.md
5. **通知弟子** - 返回决策结果

## 决策记录格式

```markdown
## [YYYY-MM-DD HH:MM] 决策记录

- **场景**: 是否继续执行任务 B
- **弟子**: task2（上等马）
- **请求**: 任务 A 已完成，是否继续任务 B？
- **决策**: 自主批准（任务 B 是原计划一部分）
- **结果**: 任务 B 已启动
```

## 异常处理

### 决策冲突
- 如果规则不明确，默认上报掌门人
- 如果时间紧急且无法联系掌门人，选择保守方案

### 决策失误
- 发现决策错误后立即上报掌门人
- 记录失误原因，更新决策规则
- 如果造成损失，启动回滚流程

## 学习机制

每次决策后：
1. 记录决策过程和结果
2. 如果掌门人事后纠正，更新规则
3. 定期（每周）回顾决策质量
4. 优化决策规则，提升准确率

## 安全边界

**绝对不能自主决策的事项**：
- 删除生产数据
- 暴露敏感信息
- 修改安全配置
- 超出预算 50% 以上的操作
- 涉及法律/合规的决策

**遇到以上情况，必须立即暂停并上报掌门人。**
