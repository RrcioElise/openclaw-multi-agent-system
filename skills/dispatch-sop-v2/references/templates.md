# templates.md

以下模板可直接复制使用。

## 1) 拆单汇报模板

```md
【拆单汇报】
任务：<任务名称>
目标：<本次要达成的结果>
截止：<时间>
验收标准：
- <标准1>
- <标准2>

子任务分配：
1. <子任务A> | owner: <姓名/agent> | 预计: <时长> | 输出: <交付物>
2. <子任务B> | owner: <姓名/agent> | 预计: <时长> | 输出: <交付物>
3. <子任务C> | owner: <姓名/agent> | 预计: <时长> | 输出: <交付物>

文件owner映射：
- <file/path/a> -> <owner>
- <file/path/b> -> <owner>

关键依赖/风险：
- <依赖或风险1>
- <依赖或风险2>

下一里程碑：<时间点 + 目标>
```

## 2) 分诊快报模板

```md
【分诊快报】
时间：<YYYY-MM-DD HH:mm>
状态：<正常/阻塞/风险上升>

已完成：
- <完成项1>
- <完成项2>

进行中：
- <进行项1>（owner: <人>，预计完成: <时间>）

阻塞项：
- <阻塞描述> | 等级: <高/中/低> | 责任方: <人/团队>

处理动作：
- <已采取动作>
- <下一动作>

是否影响SLA：<否/是，说明影响范围>
下次同步：<时间>
```

## 3) 上线 PASS / NO-GO 模板

```md
【Gate结论】
结论：<PASS / NO-GO>
时间：<YYYY-MM-DD HH:mm>
范围：<本次上线范围>

验收核对：
- [x]/[ ] <验收项1>
- [x]/[ ] <验收项2>
- [x]/[ ] <验收项3>

关键风险：
- <风险1>
- <风险2>

回退方案：
- <回退步骤或回退点>

若为NO-GO：
- 未通过原因：<原因>
- 修复owner：<人/团队>
- 预计修复完成：<时间>
- 下次Gate：<时间>

指令：<继续发布 / 暂停发布 / 回退>
```
