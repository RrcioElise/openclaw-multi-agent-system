# workflow.md

Intake → Planning → Execution → Gate 的可执行检查清单。

## 1) Intake 清单

- [ ] 明确目标、范围、截止时间、验收标准
- [ ] 判断优先级（紧急/常规/探索）
- [ ] 判断是否需要多人并行
- [ ] 30秒回执已发送

输出：任务卡片（目标、范围、验收、时限、优先级）

## 2) Planning 清单

- [ ] 拆分子任务（可独立交付）
- [ ] 分配 owner（人/文件/模块）
- [ ] 标注依赖与关键路径
- [ ] 制定同步节奏与汇报模板
- [ ] 冲突预检完成（同文件无多owner）

输出：执行计划（子任务列表 + owner映射 + 里程碑）

## 3) Execution 清单

- [ ] 每个owner只改动认领范围
- [ ] 每5-10分钟同步一次状态
- [ ] 阻塞问题即时上报
- [ ] 关键变更留痕（原因、影响、下一步）

输出：阶段性成果与风险清单

## 4) Gate 清单

- [ ] 子任务全部回收
- [ ] 验收标准逐条核对
- [ ] 风险与回退方案齐备
- [ ] 输出PASS/NO-GO结论

输出：上线结论与行动建议

## 分支处理

### A. 发现文件冲突

1. 冻结冲突文件并停止并行写入。
2. 指定唯一owner处理合并。
3. 更新owner映射并通知相关人。

### B. 发现阻塞依赖

1. 标注阻塞级别（高/中/低）。
2. 分诊并给出替代路径。
3. 重新评估里程碑与SLA承诺。

### C. Gate未通过（NO-GO）

1. 给出未通过条目与证据。
2. 给出修复清单与预计时间。
3. 约定下一次Gate时间点。
